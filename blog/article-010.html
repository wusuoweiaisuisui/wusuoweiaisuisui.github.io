用Python做期货参数优化：网格搜索和Walk-Forward验证

策略逻辑定好之后，怎么确定参数？

很多人的做法是：手动试几个，看哪个收益最高，就用哪个。这叫"暴力调参"，结果很可能是过拟合。

这篇写两种更科学的方法：网格搜索 + Walk-Forward验证。

为什么不能直接用全期最优参数

假设你的策略有两个参数：Z-score入场阈值（1.5到3.0之间）、均值回归窗口（10到60之间）。

你在2018-2024年的数据上跑了1000种参数组合，找到了最好的那组。

问题：这组参数是在这段历史数据上最优的，不代表在未来也最优。可能这段历史数据有特殊的规律，这组参数恰好适合这段数据，但实盘会失效。

这就是过拟合（Overfitting）。

网格搜索：找出稳健区间，不是最优点

网格搜索本身没问题，问题是如何使用结果。

正确用法：不要选最优参数，选稳健区间。


import pandas as pd
import numpy as np
from itertools import product

def run_backtest(price_a, price_b, z_entry, window):
    """简化回测函数，返回夏普比率"""
    spread = price_a - price_b
    
    mean = spread.rolling(window).mean()
    std = spread.rolling(window).std()
    z = (spread - mean) / std
    
    signal = pd.Series(0, index=spread.index)
    signal[z > z_entry] = -1   # 反套
    signal[z < -z_entry] = 1    # 正套
    signal[z.abs() < 0.5] = 0   # 平仓
    
    position = signal.shift(1).fillna(0)
    pnl = position * spread.diff()
    
    if pnl.std() == 0:
        return 0
    
    sharpe = np.sqrt(252) * pnl.mean() / pnl.std()
    return sharpe

# 参数网格
z_entry_range = [1.5, 1.8, 2.0, 2.2, 2.5, 2.8, 3.0]
window_range = [10, 15, 20, 25, 30, 40, 60]

results = []
for z_entry, window in product(z_entry_range, window_range):
    sharpe = run_backtest(price_a, price_b, z_entry, window)
    results.append({"z_entry": z_entry, "window": window, "sharpe": sharpe})

results_df = pd.DataFrame(results)

# 关键：不只看最大值，看稳健区间
pivot = results_df.pivot(index="z_entry", columns="window", values="sharpe")
print(pivot.round(2))

输出一个参数热力图，然后找夏普比率都还不错的区域（不是最高点，是一片高值区）。

在这片区域里，任意一组参数表现都差不多，说明策略对参数不敏感——这才是稳健的策略。

Walk-Forward验证：模拟真实交易的时序

Walk-Forward的核心思想：用过去的数据训练参数，然后在未来的数据上测试，依次滚动。


def walk_forward_test(price_a, price_b, 
                       train_months=12, test_months=3,
                       z_range=[1.5, 2.0, 2.5],
                       window_range=[15, 20, 30]):
    """
    每次用12个月数据找最优参数，在接下来3个月测试
    """
    results = []
    
    total_months = len(price_a) // 21  # 约21个交易日/月
    
    start = 0
    while start + train_months * 21 + test_months * 21 <= len(price_a):
        train_end = start + train_months * 21
        test_end = train_end + test_months * 21
        
        # 训练集：找最优参数
        train_a = price_a.iloc[start:train_end]
        train_b = price_b.iloc[start:train_end]
        
        best_sharpe = -999
        best_z, best_w = 2.0, 20  # 默认值
        
        for z, w in product(z_range, window_range):
            s = run_backtest(train_a, train_b, z, w)
            if s > best_sharpe:
                best_sharpe = s
                best_z, best_w = z, w
        
        # 测试集：用训练集最优参数测试
        test_a = price_a.iloc[train_end:test_end]
        test_b = price_b.iloc[train_end:test_end]
        
        test_sharpe = run_backtest(test_a, test_b, best_z, best_w)
        
        results.append({
            "train_period": f"{start}-{train_end}",
            "best_z": best_z,
            "best_window": best_w,
            "train_sharpe": best_sharpe,
            "test_sharpe": test_sharpe
        })
        
        # 滚动：向前推进test_months个月
        start += test_months * 21
    
    return pd.DataFrame(results)

wf_results = walk_forward_test(price_a, price_b)
print(wf_results)
print(f"\n训练平均夏普: {wf_results['train_sharpe'].mean():.2f}")
print(f"测试平均夏普: {wf_results['test_sharpe'].mean():.2f}")
print(f"衰减比例: {wf_results['test_sharpe'].mean()/wf_results['train_sharpe'].mean():.1%}")

如何解读Walk-Forward结果

关键指标：训练夏普 vs 测试夏普的比率（衰减比例）

  • 测试/训练 > 70%：优秀，策略稳健
  • 测试/训练 50%-70%：可接受
  • 测试/训练 < 50%：过拟合风险高，慎用

另外看测试期间的分布：是否大多数测试期间都是正夏普？如果有几个测试期间大亏，说明策略有脆弱的边界条件。
