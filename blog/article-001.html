什么叫"黑色系套利"

黑色系这个叫法，来自钢铁产业链的颜色——铁矿石、焦炭、焦煤、螺纹钢、热轧卷板，这一串品种都在大商所和上期所交易。因为它们在产业链上有强关联，所以价差往往具有均值回归的特性。

但"往往"不等于"一定"。

2021年焦煤一路暴涨，螺纹钢没跟上，导致我的正套仓位扛了三周。那段时间账户浮亏7%，每天盯着屏幕难受。最后Z-score确实回归了，但期间的煎熬远超预期。

这是我学到的第一课：价差均值回归，但时间可以很长。

协整关系是核心逻辑

套利的数学基础是协整检验（Cointegration Test）。两个品种的对数价格序列，如果存在协整关系，那它们的线性组合是平稳的——也就是说，价差会在某个均值附近波动，不会无限发散。

用Python做这个检验很简单：


from statsmodels.tsa.stattools import coint
import numpy as np

# price_a, price_b 是两个品种的收盘价序列
score, pvalue, _ = coint(np.log(price_a), np.log(price_b))
if pvalue < 0.05:
    print(f"协整成立，p={pvalue:.4f}")

p值小于0.05，通常认为协整关系显著。但要注意——协整检验本身就有局限性，样本外失效是常见问题。我每隔3个月重新跑一次检验，看关系是否还稳定。

Z-score策略的实际操作

协整确认后，计算价差的Z-score来决定入场：


Z = (当前价差 - 历史均值) / 历史标准差

当Z > 2（价差异常偏高）→ 反套（空A买B）

当Z < -2（价差异常偏低）→ 正套（买A空B）

当Z回归到0附近 → 止盈平仓

听起来简单，实际不是。

三个坑我都踩过

第一个坑：用收盘价计算，实盘用开盘价成交

回测爽，实盘一看，价差已经跑了100个点。原因是日线收盘价和次日开盘之间有跳空，用前一天的Z-score开盘就进，滑点巨大。

后来改成等市场稳定10分钟后再看信号。

第二个坑：两条腿不同时成交

期货套利不能"一条腿先下单，等成交再下另一条"——这叫腿差风险。我有一次螺纹钢先买进去了，热卷突然跌停，卖不出去，那次扛了整整一天。

现在用TqSdk的目标持仓API，两腿同步下单，自动处理滑点。

第三个坑：多空方向搞反了

这个说出来有点丢人。反套是"价差偏高时卖A买B"，但我代码里写的通知消息是固定"买A卖B"——结果通知说正套，实际下的是反套仓位，搞了一段时间才发现。

教训：一定要对着实盘持仓截图校验通知消息，不能光看代码逻辑。

后记

有人问我，现在做黑色系套利还有机会吗？我的判断是：市场越来越难，但机会还在。产业链关联依然存在，机构化进程让价格更有效但不会完全消灭价差机会。

关键在于：你得有稳定的技术框架，有风控纪律，还要能承受盘中的煎熬。
