2. 群设置 → 机器人 → 添加机器人 → 自定义

4. 复制webhook地址

第二步：测试推送


import requests
import json

webhook = "https://oapi.dingtalk.com/robot/send?access_token=你的token"

def send_dingtalk(title, content):
    headers = {"Content-Type": "application/json"}
    data = {
        "msgtype": "markdown",
        "markdown": {
            "title": title,
            "text": f"## {title}\n\n{content}"
        }
    }
    resp = requests.post(webhook, headers=headers, data=json.dumps(data))
    return resp.json()

# 测试
result = send_dingtalk("交易测试", "如果收到这条消息，说明配置成功")
print(result)

如果返回 {'errcode': 0, 'errmsg': 'ok'}，配置成功。

注意关键词限制：消息里必须包含你设置的关键词，否则钉钉会拒绝。我设的"交易"，所以每条推送消息里都要带"交易"两个字。

期货交易推送模板


def push_trade_signal(strategy, instrument, action, direction, detail_msg):
    """
    strategy: 策略名称
    instrument: 品种名称  
    action: 开仓/平仓/止损
    direction: 正套/反套/做多/做空
    detail_msg: 详细信息
    """
    title = f"交易信号 | {strategy}"
    
    # 动作图标
    icons = {"开仓": "🟢", "平仓": "🔵", "止损": "🔴", "止盈": "⚪"}
    icon = icons.get(action, "📊")
    
    text = f"""## {icon} {strategy} - {action}

**品种**: {instrument}
**操作**: {direction} | {action}

{detail_msg}

---
*{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}*"""
    
    return send_dingtalk(title, text)

容错处理

网络不稳定时，推送可能失败。加上重试和异常捕获：


import time

def push_safe(title, content, max_retry=3):
    for i in range(max_retry):
        try:
            result = send_dingtalk(title, content)
            if result.get("errcode") == 0:
                return True
            print(f"推送返回错误: {result}")
        except Exception as e:
            print(f"推送失败({i+1}/{max_retry}): {e}")
        time.sleep(2)
    
    # 推送失败也不能让主程序崩溃
    print(f"WARNING: 推送最终失败，消息: {title}")
    return False

这里有个重要原则：推送失败不能让主交易程序崩溃。所有推送调用都放在try-except里。

一个真实的踩坑

我有一段时间出现了"重复推送"的问题——同一笔交易收到了两三条相同的通知。

排查了半天，原因是：服务器重启后，旧的Python进程没有退出，新进程也启动了，两个进程同时在跑，同时收到信号，同时推送。

解决方案：启动前先kill旧进程


# 在启动脚本里加上这段
import os
import subprocess

def kill_existing():
    result = subprocess.run(
        ["tasklist", "/FI", "IMAGENAME eq python.exe"],
        capture_output=True, text=True
    )
    current_pid = str(os.getpid())
    for line in result.stdout.split("\n"):
        if "python.exe" in line:
            pid = line.split()[1]
            if pid != current_pid:
                print(f"杀掉旧进程 PID: {pid}")
                subprocess.run(["taskkill", "/F", "/PID", pid])

推送内容的设计原则

做了几年交易通知系统，总结了几个原则：

1. 通知里写清楚"买什么卖什么"，不要只写方向

"开多1手" → 信息不足

"买HC2510 @3420 卖RB2510 @3580" → 清楚，核验方便

2. 包含Z-score或信号值

方便事后复盘时核对当时的进场条件是否合理。

3. 止损通知要突出

止损和开仓的图标要明显区分，止损是🔴，开仓是🟢，一眼就能看出。

4. 每天收盘后发一条日报

汇总当天的开平仓记录、当前持仓、浮盈浮亏。我的日报每天17:00自动推送。
