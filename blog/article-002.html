为什么选TqSdk

国内期货程序化有几个选择：TB（Trade Blotter）、文华财经、TqSdk（天勤）。

TB和文华都是图形化为主，策略语言相对封闭。TqSdk是Python原生的，开源，可以直接用pandas、statsmodels这些熟悉的库。对我来说，选择很明确。

TqSdk另一个好处是：模拟账户和实盘账户的代码几乎一样，只是TqAccount对象传的参数不同。这让调试非常方便——先在模拟盘跑，确认逻辑没问题再切实盘。

最小可运行结构

一个TqSdk策略的骨架长这样：


from tqsdk import TqApi, TqAuth, TqAccount

api = TqApi(
    account=TqAccount("期货公司名", "账户", "密码"),
    auth=TqAuth("天勤账号", "天勤密码")
)

# 订阅数据
klines = api.get_kline_serial("KQ.m@SHFE.rb", 60)  # 螺纹钢主力，1分钟K线
quote = api.get_quote("KQ.m@SHFE.rb")

# 主循环
while True:
    api.wait_update()
    if api.is_changing(klines.iloc[-1], "datetime"):
        # 新K线出现时执行策略逻辑
        close_price = klines["close"].iloc[-1]
        print(f"最新收盘价: {close_price}")

api.wait_update() 是核心——它会阻塞程序，直到有任何数据更新（行情、账户、K线等），然后返回。这个机制非常高效，不会占用CPU轮询。

主力合约格式：这个坑我见过太多人踩

很多初学者会写成 "SHFE.rb2501" 这种格式——这是固定合约。但期货有换月问题，下月合约流动性会突然变差，你的策略可能拿着一个流动性很差的合约进出场。

用主力合约格式："KQ.m@SHFE.rb"

KQ.m@ 是天勤的主力合约前缀，会自动指向当前主力合约。换月时系统会自动切换，不需要你手动改代码。

这一个细节，省了我很多麻烦。

下单：用目标持仓，不要用直接下单

TqSdk提供两种下单方式：

方式一：直接下单


api.insert_order("KQ.m@SHFE.rb", "BUY", "OPEN", 1)

方式二：目标持仓（推荐）


pos_task = api.get_position_task("KQ.m@SHFE.rb")
pos_task.set_target_volume(1)  # 目标持仓1手多

我强烈推荐用目标持仓。原因是：目标持仓会帮你处理部分成交、拒单、撤单重下等细节。你只需要告诉系统"我要持有多少手"，剩下的它来搞定。

直接下单需要自己处理这些边界情况，代码复杂度高10倍不止。

第一个完整策略：双均线

把上面的骨架加上策略逻辑：


from tqsdk import TqApi, TqAuth, TqAccount
import numpy as np

api = TqApi(
    account=TqAccount("南华期货", "账号", "密码"),
    auth=TqAuth("你的天勤账号", "密码")
)

symbol = "KQ.m@SHFE.rb"
klines = api.get_kline_serial(symbol, 3600)  # 1小时K线
pos_task = api.get_position_task(symbol)

while True:
    api.wait_update()
    if not api.is_changing(klines.iloc[-1], "datetime"):
        continue  # 只在新K线形成时处理
    
    close = klines["close"]
    ma5 = close.iloc[-6:-1].mean()    # 5日均线
    ma20 = close.iloc[-21:-1].mean()  # 20日均线
    
    if ma5 > ma20:
        pos_task.set_target_volume(1)   # 金叉，持多1手
    elif ma5 < ma20:
        pos_task.set_target_volume(-1)  # 死叉，持空1手

这就是一个完整的双均线策略，大概40行代码。逻辑清晰，没有废话。

模拟回测 vs 真实回测

TqSdk有内置的回测模块TqBacktest，可以在本地回放历史行情：


from tqsdk import TqBacktest
from datetime import date

api = TqApi(
    backtest=TqBacktest(
        start_dt=date(2023, 1, 1),
        end_dt=date(2024, 1, 1)
    ),
    auth=TqAuth("账号", "密码")
)

但我要说一个真实情况：TqBacktest需要网络连接下载历史数据，速度和稳定性有时候不太好。遇到超时就用本地缓存，或者用akshare先下载好数据做离线回测。

部署上线之前要做的事

策略逻辑调通后，部署到生产环境前，我的检查清单：

1. 检查主力合约格式：用 KQ.m@SHFE.rb 不要用固定月份

2. 检查下单方向：在代码里加 print(f"当前持仓目标: {target}") 确认方向

3. 检查风控条件：最大持仓上限、交易时间判断（避免夜盘结束前乱下单）

4. 钉钉推送：下单/平仓时推送通知，人不在电脑前也能第一时间知道

第4点尤其重要。我的系统每次开仓都会推钉钉，包含：品种、方向、价格、仓位。有一次通知和实际方向不符，正是这个通知让我及时发现了代码bug。

总结

程序化交易的门槛，说高不高，说低不低。TqSdk把下单、持仓管理这些基础设施都封装好了，你的主要工作是：

1. 搞清楚策略逻辑

2. 写好风控

3. 跑通模拟

4. 监控实盘

代码本身不难，难的是策略本身，还有实盘中的心态。
